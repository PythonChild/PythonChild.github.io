（C/C++篇）谈谈const与#define
2019-07-26

一、两者的定义

  • const: const是一个C语言（ANSI C）的关键字，它限定一个变量不允许被改变，产生静态作用，使用const在一定程度上可以提高程序的安全性和可靠性（不能被用户更改）。
  • #define：#define是一个无参宏定义，#define即可以定义常量，也可以定义一个具体的函数（下文会详细描述）。

  • 作用阶段：从上述代码，我们可以看出，使用#define时，前面还有一个”#”，这是一个预处理符号，表明该语句是在编译的预处理阶段就起作用了，无法进行调试，因为在预编译阶段就已经替换掉了；const是伴随变量的定义及函数的定义等等，所以它是在编译期、运行期才发挥作用，而且可以进行调试。

  • 作用方式：由于#define发生在预处理阶段，也就是编译之前，它只会进行简单的字符替换，而不进行任何类型检查；而由于const在编译期有效，而且const是有对应的数据类型的，所以它会进行类型检查，避免一些低级错误。

  • 重定义：#define可以通过#undef来取消某个符号的定义，进行重新定义，但const不可以。

  • 数据存储空间：通过#define定义的符号常量，占用的是代码段中；const修饰的数据，本质还是这个数据，只是不允许更改，因此还是存放在数据段中。

  • 存储方式：#define只是进行字符的等效替换，出现多少次，就替换多少次，因此它定义的宏常量在内存中有多份备份，而const定义的常量在程序的生存周期中，一般只有一份备份，两者对空间资源的占用相差比较大。

三、总结

  • 从以上的各个点上，我们可以清晰的看出const和#define两者的各项区别，虽然看起来#define相对于const来讲，会存在诸多问题，但存在即合理，某些时候，#define也是无可替代的，实际开发也不可能完全不使用#define，但引用《Effective C++》的话，在可以使用const的情况下，尽量避免使用#define。
